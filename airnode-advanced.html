⚡ How USB mode works

No WiFi, no NodeMCU needed. The Arduino Uno plugs directly into your PC via USB. A small Python script reads the serial data and serves it as a local web API on localhost:8080. This dashboard fetches from that address.

💡 Everything runs on your PC — no internet, no router, no second board.

🐍 Install Python & pyserial

You need Python 3 and the pyserial library. Open a terminal (Command Prompt on Windows) and run:

pip install pyserial

1️⃣ Arduino Uno sketch

Upload AirNode_Uno_USB.ino to the Uno. Select board: Arduino Uno, correct COM port, then Upload.

AIRNODE_START
24.00,61.00,342,green
24.00,61.00,338,green
⚠️ Close Serial Monitor before running the Python bridge.

2️⃣ Run the Python bridge

Close Arduino Serial Monitor. Then in a terminal run:

python airnode_bridge.py
💡 Leave this terminal open — it must keep running while you use the dashboard.

3️⃣ Open the dashboard

Open this HTML file in Chrome, Firefox, or Edge. The dashboard auto-connects to localhost:8080. The status dot turns green within a few seconds.

🛡 Safety Features

Danger Lockout Overlay

When air quality exceeds the Poor threshold, a full-screen overlay blocks the UI and requires you to confirm safety steps before dismissing. A 30-second auto-dismiss countdown runs in parallel. You can disable this in Settings → Safety & Alerts.

Safety Action Panel

On any warning or danger reading, a panel appears beneath the alert banner with six action buttons: open windows, turn on fans, evacuate, check source, call help, and reset. Each button toggles to a "done" state so you can track what you've handled.

Alert Log

Every alert (warning or danger) is recorded in the Alert Log tab with a timestamp and reading value. Alerts can be individually acknowledged or bulk-cleared. The topbar badge shows unacknowledged alert count.

SOS Quick-Action Button

The 🆘 SOS Actions button in the inline alert banner immediately shows the danger lockout overlay with its full checklist.

Audio Alerts

Toggle the 🔇 button in the topbar to enable sound. A short beep plays on moderate alerts; a longer pattern plays on danger. Requires a browser interaction first (browser policy).

Browser Notifications

Click 🔔 in the topbar to enable browser push notifications. Alerts will show even if the dashboard tab is minimized.

⚠️ These are dashboard-only safety features. For life-safety applications always install dedicated gas detectors and follow local fire safety codes.

🔧 Troubleshooting

Bridge says "No serial port found" ▼
Plug the Uno into the PC first, then run the bridge. Set SERIAL_PORT = "COM3" (Windows) or "/dev/ttyUSB0" (Linux/Mac) manually if auto-detect fails.
Dashboard shows "Bridge not running" ▼
The Python bridge must be running in a terminal. If it crashed, re-run python airnode_bridge.py. Check no firewall is blocking localhost:8080.
Bridge runs but shows garbled data ▼
Baud rate mismatch. The Uno sketch sends at 9600. The bridge listens at 9600. Change both to match.
Serial port in use / access denied ▼
Close Arduino Serial Monitor — it holds the port exclusively. Only one program can use the serial port at a time.
DHT reading failed in Serial Monitor ▼
Make sure the sketch uses #define DHT_TYPE DHT11. Check the 10kΩ pull-up resistor on the DATA pin. VCC must be 5V.
pip is not recognised ▼
Python is not on your PATH. Download from python.org, tick "Add Python to PATH" during install, then restart your terminal.
